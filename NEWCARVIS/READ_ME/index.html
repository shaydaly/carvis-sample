My Sample App

This download package contains an Android Studio project for a fully functioning Mobile App, which uses your configured AWS resources and identifiers, to demonstrate the project features you've selected. You can use this project as a starting point for your own product development, or you can follow the instructions below to copy and paste the required code into your own product.

IMPORTANT: This package was custom generated for you, using your selected project options. It contains your own resource and account identifiers and should be kept private.

  • Setup
    • Integrated Development Environment

      • 1. System Requirements


        Your system must include the following:
          - Android Studio 1.4 or newer
          - Android SDK 4.4 (KitKat) API Level 19 or newer
          - Android SDK Build-tools 23.0.1
          - Android Device with Android OS 4.0.3 (IceCreamSandwich) API Level 15 or newer

      • 2. Install Android Studio


        If you have not already installed the Android Studio IDE (Integrated Development Environment) on your host, you can download it from the following location:
        https://developer.android.com/sdk


      • 3. Install Android SDK (Software Development Kit)


        If you have not already installed the latest Android SDK, you can do so within Android Studio.

        Go to...

        Tools -> Android -> SDK Manager -> Appearance & Behavior -> System Settings -> Android SDK


        Then, select the Android SDK Platform version you would like to use. The SDK Manager will download the appropriate Android SDK Platform version and apply any available updates.

      • 4. Install AVD (Android Virtual Device)


        If you have not already installed an Android Virtual Device, you can do so within Android Studio.

        Go to...

        Tools -> Android -> AVD Manager


        Then, click "+ Create Virtual Device" and select the device you would like to use in the Android emulator (e.g., Nexus 5). Select the appropriate image from the list (e.g., Lollipop API Level 22 x86 Android 5.1.1) and click the "Next" button. Name the AVD and click the "Finish" button.

    • Facebook Sign-in

      • 5. Add Android Platform to Facebook App


        In order for Facebook Sign-in to work properly, you will need to add the Android platform to your Facebook App in the Facebook developer console. Log in to the Facebook developer console and select your app under "My Apps" menu option. Select "Settings" from the left menu. Scroll down and click on the "+ Add Platform" button and select Android. Set the following fields:

          Package Name = com.mysampleapp
          Default Activity Class Name = com.mysampleapp.MainActivity

        When you move beyond the sample app, you will set these to match your own project, but for now these values will work to demonstrate Facebook Sign-in, in the sample app.

        You will see a warning with title "Google Play Package Name" indicating your package name is not unique. That is OK. Simply click "Use this package name" button.

      • 6. Add your Development Key Hash


        When you first build the sample app project in Android Studio, if you do not already have a debug keystore, Android Studio will create one for you. This keystore contains a key that identifies your development environment. Facebook requires the key hash from your debug keystore. To generate the development key hash you can run the following command.

        On a Mac:
          keytool -exportcert -alias androiddebugkey -keystore ~/.android/debug.keystore | openssl sha1 -binary | openssl base64

        On Windows:
          keytool -exportcert -alias androiddebugkey -keystore %HOMEPATH%.androiddebug.keystore | openssl sha1 -binary | openssl base64

        You will be prompted to enter a keystore password. If you did not create your keystore with a password, then simply hit return, and the tool will output your 28 character key hash. Copy this string and paste it into the Facebook Quick Start website section called "Add your development and release key hashes" under the "Development Key Hashes" field and click "Next" to save.

  • Compile and Run
    • Launch App

      • 7. Import MySampleApp Project into Android Studio


        Once you have downloaded and unzipped the sample app project files, please open Android Studio and choose...

        Import project (Eclipse ADT, Gradle, etc.)


        Find the folder where you extracted the zip file contents and select the "MySampleApp" sub-folder. Android Studio will import the project and build it using Gradle.

        Note that the project may have several dependencies, which Gradle will automatically retrieve from online repositories. The first time you build the app, it may take longer than subsequent incremental builds.

      • 8. Run MySampleApp in Debug Mode


        Once Gradle finishes building, you can click the "Debug app" icon to launch the app in the Android emulator.

    • User Identity - Try It!

      • 9. Demonstrate User Identity


        Launch MySampleApp in the Android emulator. If you've selected to enable User Sign-in in your project, you should sign in to the app. Then, click on the "User Sign-In" demonstration section and then click the "Demo User Identity" button. You will see a unique identifier for the user. This identifies the user to AWS systems and provides the user temporary permissions to access your configured AWS resources.

      • 10. Verify User Identity


        Go to the AWS Cognito console, and you should see the count of users in your Cognito Identity Pool is now greater than zero.
        https://console.aws.amazon.com/cognito/

This section simply provides information about the generated code in the MySampleApp project.

        

The Identity Manager keeps track of the identity of the current app user by storing the user's Amazon Cognito Identity ID. It is partially responsible for signing the user in to the app if you have User Sign-in enabled in your project. 52. Facebook "App Not Setup" error message

        

If you are getting the message "App Not Setup" after a successful login it means that your Facebook App is still in development mode. For testing purposes, you can add test users to your Facebook App via the Facebook Developer Console. 
        To do that create a test user for your Facebook development application:
        1. Go to https://developers.facebook.com, log in with your developer credentials.
        2. From the "My Apps" menu, select your application used in the sample app.
        3. On the left pane, select "Roles".
        4. Under "Testers", click "Add Testers".
        5. In the Add Testers dialog, add the username of the test user (i.e. https://www.facebook.com/myuser.name.50).
        6. Click Submit.
        Now you can have that test user log into your sample app.

        For more information on adding test users go to: https://developers.facebook.com/docs/apps/test-users.

         If your app is "Live" to the General Public, then you need to provide a valid Contact Email address for your Facebook App, go to the Facebook developer portal and go to the "Settings" section of your app. Enter a valid email under "Contact Email" and click "Save changes".
